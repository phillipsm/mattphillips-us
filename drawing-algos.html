Drawing with algorithms

Drawing with algorithms is powerful - setup the guardrails and let the machine run. It doesn't grow weary.

Applications are broad. Patterns can be beautiful in print, and are a natural fit serving as backgrounds, screen art and information rich display systems.

Below, you'll find a few recent examples I made while seeking backgrounds for a wall-mounted display.

This visual grows slowly. It's an accretion of color-shifting treads.

A wobbly line is drawn across the HTML Canvas as a circle steadily moves on the x axis and bounces 80 times per second on the y axis.

In this moving image, we see a message in the noise. It scrolls. What does it say?

There are two things that get drawn to this canvas - the red horizontal lines every 10 milliseconds and a letter every one second.

Here, we have the simplest animated visual - it's one line between two random points, 80 times per second.

The p5.js random function makes it easy to pull a color from a list of colors for each line drawn.

Role

I used the JavaScript visualization library, p5.js to manage logic and to interact with the <canvas> element. The common .setup() and .draw() pattern makes it quick to spin up something interesting.
